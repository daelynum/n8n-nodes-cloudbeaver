# n8n-nodes-cloudbeaver

This is an n8n community node for [CloudBeaver](https://dbeaver.com/docs/cloudbeaver/) — a web-based database management
tool. It lets you execute SQL queries against any database connected to your CloudBeaver server directly from n8n
workflows.

[n8n](https://n8n.io/) is a [fair-code licensed](https://docs.n8n.io/sustainable-use-license/) workflow automation
platform.

## Installation

Follow the [installation guide](https://docs.n8n.io/integrations/community-nodes/installation/) in the n8n community
nodes documentation.

Or install directly in n8n: **Settings -> Community Nodes -> Install** -> `n8n-nodes-cloudbeaver`

## Operations

- **Execute SQL Query** — run any SQL query against a database connected to CloudBeaver, with support for limit, offset,
  ORDER BY, and WHERE filters

## Credentials

Create a **CloudBeaver API** credential with:

| Field               | Description                                                                                                                      |
|---------------------|----------------------------------------------------------------------------------------------------------------------------------|
| Server URL          | Base URL of your CloudBeaver server, e.g. `https://cloudbeaver.example.com`                                                      |
| Authentication Type | `API Token` (PRO) or `Local access` (Username/Password)                                                                          |
| API Token           | Token generated by Admin in CloudBeaver PRO — [how to generate](https://dbeaver.com/docs/cloudbeaver/Generate-API-access-token/) |
| Username / Password | Local CloudBeaver account credentials — [docs](https://dbeaver.com/docs/cloudbeaver/Local-Access-Authentication/)                |

> **Note:** Only connections from the **Shared project** in CloudBeaver are supported.

## Compatibility

Requires n8n 1.0.0 or later. Designed for self-hosted CloudBeaver instances (Community or PRO edition).

## Usage

### Finding the Connection ID

Right-click a database in CloudBeaver's **Database Navigator** -> **Open** -> copy the ID from the connection window (e.g.
`postgres-jdbc-19d5ca3a223-2f46e1a321049b15`).

### Query with filters

The **Where** and **Order By** fields are applied on top of your SQL query:

**SQL Query:**

```sql
SELECT id, name, email
FROM users
```

**Where:** `status = 'active'`  
**Order By:** `created_at DESC`  
**Limit:** `50`

Executes as:

```sql
SELECT *
FROM (SELECT id, name, email FROM users) AS _q
WHERE status = 'active'
ORDER BY created_at DESC LIMIT 50
```

## Getting Started (Local Testing)

1. Start CloudBeaver:

    ```bash
    docker run -d -p 8978:8978 dbeaver/cloudbeaver:latest
    ```
    
    Or follow
    the [full installation guide](https://github.com/dbeaver/cloudbeaver-deploy?tab=readme-ov-file#installation-with-docker-image).

2. Configure authentication — follow
   the [Local Access Authentication guide](https://dbeaver.com/docs/cloudbeaver/Local-Access-Authentication/). Create a
   user (login/password) or use admin credentials.

3. Create a database connection — follow
   the [Create Connection guide](https://dbeaver.com/docs/cloudbeaver/Create-Connection/) (tested with PostgreSQL).

4. Get the Connection ID — open [Database Navigator](https://dbeaver.com/docs/cloudbeaver/Database-Navigator/),
   right-click a database -> **Open**, and copy the connection ID from the opened window.

5. In n8n:
    - Create **CloudBeaver** credentials (Local auth: login/password)
    - Add the **CloudBeaver** node to your workflow
    - Set the **Connection ID**
    - Enter a SQL query (e.g. `SELECT 1`)
    - Execute the node

> **Tip:** Use CloudBeaver to generate a query — Database Navigator (context menu) -> **Generate SQL** — to quickly get a
> test SELECT, INSERT, etc.
